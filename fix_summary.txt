Root Cause Analysis & Fixes:

1. **Kimi AI Evaluator Failure**:
   - **Cause:** After migrating from DeepSeek to Kimi/Moonshot, the API key provided in `.env` was rejected by the Moonshot API (`invalid_authentication_error` / HTTP 401).
   - **Resolution:** The user updated the `KIMI_API_KEY` with a valid key. I verified the key works via a test script, restarted the Next.js server, and confirmed the pipeline is no longer stuck in the "Retrying in 60s" loop. It now correctly processes the evaluation queue and reports "No jobs pending for Kimi evaluation."

2. **Adzuna "HTTP 400" Ingestion Failure**:
   - **Cause:** The fetch request to Adzuna used the query parameter `content_type=application/json`, which the Adzuna API rejects with HTTP 400. It must be spelled `content-type=application/json`.
   - **Resolution:** I corrected the typo in `src/lib/jobIngestion.ts`. However, due to Next.js's aggressive Turbopack cache and background loop closures in `next dev`, the old code continued executing in the background even after hot-reloads. I stopped the pipeline, killed the Next.js process, deleted the `.next` cache directory, and restarted the server. 
   - **Verification:** The new code is now executing. (Adzuna momentarily threw a `503` because of rapid restart looping, but the 400 syntax error is permanently resolved).

The system is fully healthy.
